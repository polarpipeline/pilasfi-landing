PilasFi - Documentacion Tecnica

Ecosistema de Gestion Financiera Inteligente con categorizacion automatica de gastos mediante IA.

Que es PilasFi?

PilasFi es una plataforma integral de finanzas personales que automatiza el registro de gastos mediante el procesamiento inteligente de notificaciones bancarias por correo electronico. Utilizando IA avanzada (OpenAI GPT-4o, Google Gemini), la plataforma extrae automaticamente transacciones de emails bancarios y las categoriza de forma inteligente.

ℹ
Base URL de Produccion: https://api.pilasfi.com/api/v1

Microservicios

pilasfi-api Port: 8080

Core business logic, user management, transactions y analytics.

Responsabilidades:

  • Autenticacion y autorizacion (Firebase)
  • API REST para transactions, budgets, analytics
  • Gestion de usuarios y households
  • Webhooks para Pub/Sub (inbox pattern)
  • Push notifications (FCM)
  • Crowdsourced merchant categorization

pilasfi-email-sync Port: 8081

Sincronizacion de emails desde Gmail/Outlook y gestion de tokens OAuth.

Responsabilidades:

  • Consumir eventos de configuracion de email
  • Refrescar tokens OAuth automaticamente
  • Fetch de emails crudos desde Gmail/Outlook
  • Normalizacion y limpieza de HTML (Jsoup)
  • Publicacion de emails a email-raw topic
  • Distributed locks para prevent duplicate processing
pilasfi-parser-service Port: 8082

Parseo de emails bancarios y extraccion de transacciones con IA.

Responsabilidades:

  • Consumir raw emails del topic email-raw
  • Detectar banco basado en remitente
  • Parseo deterministico (Regex + Jsoup)
  • Fallback a IA (OpenAI/Gemini)
  • Generar candidatos de parsers con IA
  • Publicar transacciones al topic email-transaction

Bancos Soportados:

Pichincha Produbanco Banco Guayaquil Diners Club PacifiCard Deuna Titanium + Auto-discovery (IA)
pilasfi-sync-scheduler Port: 8082

Orquestador de sincronizacion distribuida de emails.

Responsabilidades:

  • Recibir triggers de Cloud Scheduler (cada 5 min)
  • Consultar email_configs que necesitan sincronizacion
  • Crear sync_jobs (outbox pattern)
  • Publicar eventos a Pub/Sub topic
  • Garantizar no duplicacion con locks atomicos

Flujo de Datos

Flujo de Procesamiento de Emails

1

Ingesta

Los bancos envian notificaciones -> Gmail/Outlook -> Usuario conecta cuenta via OAuth en pilasfi-api

2

Configuracion

pilasfi-api publica EmailConfigUpdatedEvent al topic user-email-config

3

Sincronizacion

pilasfi-email-sync consume el evento, hace fetch de emails desde Gmail/Outlook, limpia HTML y publica a email-raw topic

4

Parsing

pilasfi-parser-service consume emails, intenta parseo deterministico, si falla usa IA (OpenAI/Gemini), publica transacciones a email-transaction topic

5

Persistencia

pilasfi-api consume transacciones, las guarda en DB, actualiza cache Redis, envia push notification via FCM

6

Presentacion

App movil recibe notificacion, usuario puede categorizar si es necesario, categorizacion se propaga a todos los usuarios (crowdsourcing)

Arquitectura Event-Driven

Topics de Google Cloud Pub/Sub

Topic Publisher Consumer Descripcion
user-email-config pilasfi-api email-sync Configuracion de email actualizada
email-raw email-sync parser-service Emails crudos para parsear
email-transaction parser-service pilasfi-api Transacciones parseadas
token-refreshed email-sync pilasfi-api Token OAuth refrescado
sync-email-request sync-scheduler email-sync Request de sincronizacion

Parsers de Bancos

Flujo de Parsing

RawEmail (email-raw topic) | v Normalizacion (HTML cleaning con Jsoup) | v Remitente conocido? | +-- SI --> BankParserFactory | +-- PichinchaParser | +-- ProdubancoParser | +-- GuayaquilParser | +-- DinersClubParser | +-- PacifiCardParser | +-- DeunaParser | | | v | ParsedTransaction | +-- NO --> AIParserService +-- OpenAI GPT-4o / Google Gemini +-- Extract: amount, date, merchant, type +-- Generate regex patterns | +-- ParserCandidate (si confidence > 0.90 -> promover a parser) | v TransactionProducer --> email-transaction topic

Seguridad

Firebase Authentication

  • Metodo: JWT tokens
  • Flujo: Usuario login via Google/Email en app -> Firebase emite ID token -> App envia token en Authorization header -> pilasfi-api verifica con Firebase
  • Endpoints publicos: /q/health, /q/swagger-ui

OAuth Integration

Gmail (Google)

  • Scopes: gmail.readonly, userinfo.email
  • Tokens almacenados encriptados en DB
  • Refresh automatico

Outlook (Microsoft)

  • Scopes: Mail.Read, User.Read
  • Tokens almacenados encriptados en DB
  • Refresh automatico

Secrets Management

  • Produccion: Google Cloud Secret Manager, sealed-secrets en K8s
  • Development: .env file (NO committed)
  • Encryption: AES-256 para datos sensibles
